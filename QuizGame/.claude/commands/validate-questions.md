# validate-questions

`src/data/questions.ts`의 문제를 CLAUDE.md 가이드라인에 따라 검증합니다.  
카테고리를 지정하면 해당 카테고리만, 생략하면 전체 문제를 검증합니다.

## 사용법

```
/validate-questions [카테고리]
```

예:
- `/validate-questions` — 전체 문제 검증
- `/validate-questions 과학` — 과학 카테고리만 검증

## 절차

1. `src/data/questions.ts`를 읽어 문제 목록을 파악한다.
2. `$ARGUMENTS`가 있으면 해당 카테고리(`한국사` | `과학` | `지리` | `일반상식`) 문제만 대상으로 하고,  
   없으면 전체 문제를 대상으로 한다.  
   유효하지 않은 카테고리가 입력된 경우 "알 수 없는 카테고리입니다" 를 출력하고 중단한다.
3. 각 문제에 대해 아래 항목을 순서대로 점검한다:

   **구조 검사**
   - **중복 ID**: id 값이 전체 문제에서 유일한가
   - **answer 범위**: `0 | 1 | 2 | 3` 범위 내인가
   - **choices 수**: 정확히 4개인가

   **내용 검사**
   - **모호한 표현**: 문제 텍스트에 아래 표현이 있으면 어떤 기준을 명시해야 하는지 구체적으로 제안한다
     - `가장` → 측정 기준(예: 면적, 인구, 높이 등)을 명시해야 함
     - `최초` → 기준 범위(예: 한국 최초, 세계 최초)와 시점을 명시해야 함
     - `최대` / `최소` → 무엇을 기준으로 최대·최소인지 명시해야 함
     - `제일` → `가장`과 동일하게 측정 기준 명시 필요
     - `유일` → 범위(국내/세계)와 시점을 명시해야 함
   - **시간·범위 명확성**: 변할 수 있는 정보(인구, 현직, 현재 등)에 시점이 있는가
   - **정답 유일성**: 정답 외 다른 보기도 정답이 될 수 있는 여지가 있는가

4. 결과를 다음 형식으로 출력한다:

   ```
   [ID 101] (과학) "가장 가벼운 원소는?"
     ⚠ '가장' — 질량 기준임을 명시 권장 (예: "원자량이 가장 작은 원소는?")

   [ID 203] (한국사) "최초의 금속 활자 인쇄본은?"
     ⚠ '최초' — "현존하는 세계 최초" 등 범위와 시점을 명시 권장
   ```

5. 이상이 없으면 "검증 완료 — 발견된 문제 없음 (N개 문제 검사)" 를 출력한다.
